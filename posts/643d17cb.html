Lark2026-03-082026-03-24

信息搜集

ip: nmap扫描一下

把cctv.htb写进/etc/hosts 访问之后使用admin/admin成功登录 扫一下目录

发现这个/api接口，再去扫一下/api接口下的目录 没扫到什么，但是在http://cctv.htb/zm/?view=options&tab=users页面发现三个用户

通过查这个版本的漏洞发现CVE-2024-51482

由于是时间盲注，所以很慢很慢，最终爆破得到的是mark:opensesame 直接ssh连接上去

提权

发现没啥权限，信息收集一波

端口 绑定地址 服务 说明
22 0.0.0.0 SSH 标准 SSH 远程登录
53 127.0.0.53/54 systemd-resolved 本地 DNS 解析服务
80 * (所有接口) Apache/Nginx ZoneMinder Web 界面
3306 127.0.0.1 MySQL/MariaDB ZoneMinder 数据库
33060 127.0.0.1 MySQL X Protocol MySQL 8.0+ 新协议

8888 127.0.0.1 ZoneMinder 控制端口 内部控制接口
8889/8999? 127.0.0.1 7999 ZoneMinder 备用/日志服务
9081 127.0.0.1 ZoneMinder API/Stream 视频流 API 端点
1935 127.0.0.1 RTMP 实时消息传输协议（流媒体）
这是每个端口对应的服务，再去翻翻配置文件，查看 systemd 服务文件

启动时加载配置文件：/etc/motioneye/motioneye.conf
查看 motion 服务配置

这里有admin用户的密码，需要将8765端口转发出来
