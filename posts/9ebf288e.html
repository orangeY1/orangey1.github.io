信息搜集

看一眼mac地址：08:00:27:36:6e:6f使用arp-scan -l确定ip地址

ip地址为<INTERNAL_IP>使用nmap扫描一下端口情况

确定22和80端口开放，先看一眼 80 端口，发现是 WordPress 博客，于是先用 wpscan 做插件和用户枚举。

80端口渗透-user flag

这里直接使用 wpscan --url http://<INTERNAL_IP> -e u,ap --plugins-detection aggressive。

扫出来2个用户名和几个插件，这个wp-query-console插件应该是有漏洞的，找一下，找到了CVE-2024-50498这个关于插件的漏洞

页面返回的结果发现了disable_functions

发现反引号能执行命令，尝试反弹一个shell

确认已经成功反弹 shell 后，再补一个交互式 TTY 方便后续操作。

拿到的是www-data的权限

还有一个welcome的用户，猜测要先提权到welcome用户，最后再拿root

找到了配置文件的一些信息

尝试连接进去找一下用户的账号密码

拿到 welcome 的口令后，直接切换到该用户查看 flag。

先拿下user’s flag，接着找提权点进行提权

提权

尝试看了一下这个文件的内容，根本不是人读的，搜了一下才发现gobuster是一款用于目录和文件枚举的开源工具，简单看了一下使用教程(https://blog.csdn.net/2302_82189125/article/details/135999920)

我一开始尝试直接用 gobuster 去碰 root.txt，但这条思路没有产出。

貌似rootflag的名字不是root.txt，下面就没有什么思路了，看了眼大佬的博客，他们用到了一个pspy64这个工具（ pspy是一个命令行工具，它可以在没有Root权限的情况下，监控Linux进程。）

把pspy64传到kali,在自己的kali上起个服务，把pspy64下载到靶机中

这是运行 pspy64 之后的一部分，可以看到 root 会定时执行 /opt/.test.sh，于是继续围绕这个脚本想办法。

看不了一点

继续利用 gobuster 把 /opt/.test.sh 当作字典文件去探测外部服务，结果如下：

这一步的核心思路是：把 /opt/.test.sh 当作 gobuster 的字典文件，再把 -u 指向自己控制的 HTTP 服务。这样靶机访问我们服务时，请求路径里就会泄露该脚本的内容。

他里面有个echo test的定时任务，然后下面的步骤是

可以在自己的kali中新建一个文件夹，然后创建一个a，再起个服务，然后返回靶机运行gobuster的指令

返回/a的原因是我们关闭了状态码的显示并使用-q开启了安静模式，因此返回的结果就是所扫描到的目录路径

(前提是kali上开启服务的文件夹中有a这个文件) 这样就能拼接字符了，既然能够拼接字符，那么可以尝试一下拼接出一个反弹shell的命令，还是在kali上的那个文件夹中，新建一个tmp文件夹(因为我们在/tmp文件夹下)，然后文件夹下面有一个b文件

这一步的目的是拼接出 /tmp/b 这个路径，其中 b 文件里保存的是反弹 shell 命令。随后在原来的 gobuster 命令后面加上 -o /opt/.test.sh，就能把扫描结果写回定时任务脚本；等 root 的定时任务再次执行时，就会触发我们写进去的命令。

可以看到已经监听到了，下面直接拿root flag即可

小结

这个靶场，先从外网打点，还是挺好打的，因为是wordpress搭建的博客系统，先使用wpscan扫描一下发现有一些插件，可以搜到插件的漏洞，有现成的poc，直接利用就好了，可以执行命令，直接反弹shell，但是发现有其他2个用户，既然是wordpress了，而且还有配置文件，就能拿到数据库的账密，连进去找一下welcome的密码，登录进welcome用户拿到user flag，下面提权就是围绕gobuster这个工具可以读取文件的内容，也可以写入内容到指定文件，这样就能把反弹shell的命令写到定时文件中，其中还有一个无需root权限就能查看定时任务的工具pspy64,这样一来，只要定时任务被执行，就能拿到root权限的shell，达到提权的效果
