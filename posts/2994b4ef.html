HMV-SingDanceRap靶机渗透记录

Lark2026-03-242026-03-24

信息搜集

ip地址为<INTERNAL_IP>

使用gobuster对目录进行扫描

针对<command>这个目录再次扫描

发现有个登录页面，但是没发现存在弱口令，猜测是sql注入，抓个数据包让sqlmap跑一下

看不懂就扔给ai，这个返回结果是说明扫描到了sql注入漏洞并且是时间盲注漏洞，接下来使用sqlmap继续跑

但是没法直接登录，显示<command>

获取 User Flag

这条路通不了，换一条路，回到news.php进行联合注入

使用命令<command>

读取文件

接着把其他文件也尝试读取出来，比如读取配置文件，因为中间件是apache，所以尝试读取apache的配置文件

可以看到网站的一些源码都在<command>目录，然后可以去读取manager.php的内容

发现可以命令执行，需要用户名是he110wor1d_admin，然后post传参command进行命令执行，但是没成功，再去读取一下login.php的源码

$sql = "SELECT id, username, password <REDACTED_PASSWORD> users where username='$username'";通过这句话可以知道是单引号闭合的，可以构造payload

这样的话，sql语句就变成了$sql = "SELECT id, username, password <REDACTED_PASSWORD> users where username='' union select 1,'he110wor1d_admin','pass'";进去之后就有一个执行命令的输入框，可以反弹shell，拿到shell之后切换到he110wor1d用户，口令已脱敏i_love_sing_dance_rap

提权

在home目录下看到一个thekey2root目录，里面有一个thekey2root文件，cat了一下，发现没法查看，看了下内容，应该是一个二进制文件，把他搞下来看看

找了hyh大佬的wp，用了他的脚本

可以看到成功进入到了 sing_dance_rap 函数，现在要想办法控制 system 的参数由于 input 函数中的数组是在函数内部声明的，也就是一个局部变量，外部无法访问到,因此我们要寻找可以写入或者能利用的字符串，很遗憾并不存在，那就只能去查找现有的字符串了，C 语言中，如果是以字符串地址作为参数的话，那么函数会按照字节逐个读取，直到遇到 \0 才停止，而在上面可以看到的是在 0x804a03e 这个位置，也就是 s 的地址，后面刚好有一个 \x00**，**有了这个合法的字符串，接下来要做的就是体现在 linux 里面，因为实际上函数内执行的是 system(“s”)，那么如果存在有一个 s 的二进制文件，而里面刚好有提权部分的代码，就可以执行命令，在本地调试一下，也是看了hyh大佬的wp，跟着一步步复现出来了

这里有很多涉及到二进制相关的内容，所以二进制相关的知识也要多看看
