HMV-Easypwn靶机渗透记录

Lark2026-03-242026-03-24

信息搜集

ip地址为<INTERNAL_IP>

搜集一下端口信息

看到这个6666端口的服务去搜了一下tcpwrapped，发现可以利用一个漏洞，但是没有成功

用dirsearch扫目录发现啥也没扫到，换成gobuster

发现一个mysecret.txt，curl一下

再curl一下

把这些当作字典，对目录进行模糊测试

测试到一个<command>目录，访问之后下载了一个文件，file了一下发现是一个压缩包，但是unzip需要密码

使用zip2john更快更方便，但是得到的文件好像是一个二进制文件

获取 User Flag

用ida打开并找到main函数，把main的代码扔给deepseek看一眼

发现是有被禁止的字符的，可以在ida里面对forbidden_bytes进行跟踪

可以发现禁用了0x00 (NULL)`` 0x20(space)``0x0F``0xCD，利用msfvenom生成一个反弹shell的shellcode

使用xxd命令可以查看文件的十六进制

成功反弹shell，然后就可以拿user flag了

提权

先把自己公钥写进去看看能不能连上

接下来可以使用dpkg -v来查看一下哪些文件被修改过

可以发现/etc/sudoers被修改了，所以现在我们使用sudo要输入密码，但是我们没有密码，这时候还得回归那个提示，需要getroot，先看一下这个getroot文件在哪

找到这个getroot，把他搞到kali里，然后在搞到本地进行分析一波

不过这时还没有 lamb 的登录口令，所以接下来要做的是继续找与该用户相关的敏感文件和线索。

lamb@pwnding:~$ find / -user lamb 2>/dev/null|grep -Pv ‘sys|proc|run’
/dev/pts/1
/home/lamb
/home/lamb/.viminfo
/home/lamb/getroot
/home/lamb/.profile
/home/lamb/.bashrc
/home/lamb/use3e3e3e3e3sr.txt
/home/lamb/.bash_logout
/home/lamb/.ssh
/home/lamb/.ssh/authorized_keys
/home/lamb/.bash_history

最后在备份目录下发现了一条更深的隐藏路径：

得到一个提示，根据上一个提示的cupp，应该是要使用cupp这个工具来生成一个字典

下面需要使用Suforce这个工具和david.txt这个字典

得到口令已脱敏，这个就是lamb用户的密码

lamb@pwnding:/tmp$ sudo -l
[sudo] password lamb:
Matching Defaults entries for lamb on pwnding:
env_reset, mail_badpass, secure_path=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin

User lamb may run the following commands on pwnding:
(ALL : ALL) PASSWD: /usr/local/bin/getroot
lamb@pwnding:/tmp$ sudo /usr/local/bin/getroot
Usage: /usr/local/bin/getroot

这是deepseek的解释，直接让deepseek帮我们写一个生成一个输出只有数字的代码，并给我们编译方案

deepseek生成的代码有问题使用了大佬的方案给出了root的密码

lamb@pwnding:$ vi a.c
lamb@pwnding:$ g++ -o key a.c -B /usr/bin/
lamb@pwnding:$ ./key
59121
lamb@pwnding:$ ./key
67002
lamb@pwnding:~$ sudo /usr/local/bin/getroot $(./key)
$1$BvrTqWyB$Soa7qkeu1GfIoy2duf53t0
